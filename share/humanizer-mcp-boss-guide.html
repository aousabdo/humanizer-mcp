Add Humanizer to Claude Desktop

Score your text for AI-detection risk and get line-by-line fix suggestions, right inside Claude.

What this is: A small tool that plugs into Claude Desktop. Once installed, you can paste any text into Claude and ask things like "score this for AI tells" — Claude will give you a 0–100 risk score and tell you exactly what to change.

Time: About 5 minutes. You'll do three things: install one helper tool, edit one config file, and restart Claude.

Requirements: Windows 10 or 11, Claude Desktop installed (free download from claude.ai/download). No coding required.

1Install the launcher tool (uv)

Claude needs a small helper tool to run the humanizer. Open PowerShell:

  1. Press the Windows key, type PowerShell, then press Enter.
  2. Paste this whole line into PowerShell (right-click to paste) and press Enter:
powershell -c "irm https://astral.sh/uv/install.ps1 | iex"

Wait about 10 seconds. It prints a few lines and finishes. Then close PowerShell completely and reopen it — that step is necessary so the next test works.

Verify the install by running:

uvx --version

You should see something like uv 0.5.x. If you see "command not recognized", restart Windows and try again.

2Tell Claude Desktop about humanizer

You'll edit one configuration file.

  1. Quit Claude Desktop completely (right-click the icon in the system tray → Quit).
  2. Press Windows + R to open the Run dialog.
  3. Paste this path and press Enter:
    %APPDATA%\Claude\
  4. You should see a file called claude_desktop_config.json. Right-click it → Open with → Notepad.
    If the file doesn't exist, open Notepad, then save a new file in the folder shown above. Important: in the Save dialog, set "Save as type" to "All Files", and use the exact name claude_desktop_config.json. Otherwise Notepad will save it as claude_desktop_config.json.txt and Claude won't find it.
  5. Replace the entire contents with this. (If the file already had other entries under mcpServers, just add the humanizer block alongside them — don't delete what was there.)
{
  "mcpServers": {
    "humanizer": {
      "command": "uvx",
      "args": ["humanizer-mcp"]
    }
  }
}

Save the file (Ctrl + S) and close Notepad.

3Reopen Claude Desktop

Launch Claude Desktop. Start a new chat. Look for the 🔨 hammer icon near the message box at the bottom — click it and you should see five tools listed:

  • humanizer_analyze_ai_tells
  • humanizer_quick_vocab_scan
  • humanizer_get_rewrite_instructions
  • humanizer_compare_before_after
  • humanizer_get_banned_words

If they're there, you're done. The first time you actually use one, Claude pauses for ~30 seconds while the tool finishes downloading in the background — after that, it's instant.

4Try it out

In any Claude chat, paste a paragraph and ask:

"Score this for AI tells and tell me what to change."

Claude returns a 0–100 risk score, the specific signals that fired (vocabulary hits, sentence rhythm, paragraph uniformity, em dashes, etc.), and a line-by-line list of what to replace. The eight-signal scoring approach is documented in the project README.

Other prompts that work:

  • "Run a quick vocab scan on this paragraph."
  • "Give me rewrite instructions for this academic abstract — keep it formal but fix the burstiness."
  • "Compare these two drafts. Did my edit lower the AI-detection risk?"

If something goes wrong

No hammer icon shows up in Claude Desktop

The JSON file probably has a small syntax error (missing comma, extra brace). Open PowerShell and run: Get-Content "$env:APPDATA\Claude\claude_desktop_config.json" | ConvertFrom-Json. If it errors, the message points at the problem line. Fix it, save, restart Claude.

Hammer icon shows but says "0 tools available"

Claude can't find uvx. In PowerShell run where.exe uvx. If it prints nothing, the install in Step 1 didn't take — repeat Step 1, restart Windows, try again.

Tools list correctly but error when used

Open %APPDATA%\Claude\logs\ and find the file beginning with mcp-server-humanizer. Send the last 20 lines to whoever set this up — they'll diagnose from there.

Notepad saved the file with a .txt extension

In File Explorer, enable "File name extensions" in the View menu, then rename the file from claude_desktop_config.json.txt to claude_desktop_config.json.
