Humanizer MCP

A Claude tool that scores your text for AI-detection risk and suggests, line by line, what to change.

Open source · MIT licensed · works in claude.ai web, Claude Desktop, and Claude for Chrome

All three give you the same five tools in Claude. The first is the easiest.

Custom Connector — paste a URL

~30 seconds No install All Claude surfaces

Best for everyone with a Claude account, including the free plan (free is limited to 1 custom connector).

  1. In Claude, click your avatar → Settings → Connectors.
  2. Click Add custom connector.
  3. Paste the URL from above into the Remote MCP server URL field.
  4. Click Save. The five humanizer_* tools appear in any chat.

Skill — upload one file

~2 minutes No install Estimated scoring

Useful if your Claude account is already at the 1-connector free-tier limit, or if you don't want a hosted URL in the loop.

  1. Download the skill zip:
    humanizer-mcp.zip (5 KB)
  2. In Claude, go to Settings → Capabilities → Skills.
  3. Click Upload skill, pick the file, and click Open.

The skill embeds the same banned-word list and scoring rubric as the MCP, but Claude estimates the math instead of computing it. Lower precision; zero infrastructure.

MCP server — install locally

~5 minutes Light technical setup Most accurate

Best for developers, or for users who want everything to run on their own machine (no data leaves the laptop).

Claude Code:

claude mcp add humanizer -- uvx humanizer-mcp
Claude Desktop config (macOS / Windows / Linux)

Edit claude_desktop_config.json and add this under mcpServers:

{
  "mcpServers": {
    "humanizer": {
      "command": "uvx",
      "args": ["humanizer-mcp"]
    }
  }
}

For a step-by-step Windows walkthrough with screenshots and troubleshooting, download the 3-page Windows guide PDF.

3. Try it

In any Claude chat, paste a paragraph and ask:

Score this for AI tells and tell me what to change.

You'll get a 0–100 risk score, the specific signals that fired, and a line-by-line list of suggested replacements. Other prompts that work:

  • "Run a quick vocab scan on this paragraph."
  • "Compare these two drafts. Did my edit lower the AI-detection risk?"
  • "Give me rewrite instructions for this academic abstract — keep it formal but fix the burstiness."

What's the difference?

The connector and the local install give you the same deterministic Python-computed scoring. The skill is a lightweight prompt-only version.

CapabilityConnector / MCPSkill
Banned-word list✓ embedded in code✓ embedded in markdown
Phrase pattern detection✓ regex✓ Claude eyeballs it
0–100 risk score✓ deterministic✗ estimated
Burstiness math✓ computed✗ estimated
Contraction ratio✓ computed✗ estimated
Em-dash count✓ exact~ approximate
Same input → same output✓✗ varies by run

How the score works

The 0–100 score combines eight signals: AI-vocabulary hits, AI-phrase hits, burstiness (sentence-length variance), contraction ratio, paragraph uniformity, rhetorical questions, first-person voice, and em-dash count. The total is bucketed into LOW (≤ 20), MEDIUM (21–50), or HIGH (51+). Full breakdown in the project README.

The server doesn't rewrite — it diagnoses and prescribes. The LLM driving your chat does the rewrite. That's the point: a planner, not a black-box laundering service.
